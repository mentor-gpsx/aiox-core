# Story 1.3: Implement Roles & Permissions Matrix

## Story Statement

Create role-based permission matrix (5 roles × 7 resources × 4 actions). Implement `@CheckPermission()` decorator for NestJS controllers to enforce access control at endpoint level.

## Roles & Permissions

| Role | Sales | Commissions | Users | Accounts |
|------|-------|-------------|-------|----------|
| ADMIN | CRUD | CRUD | CRUD | CRUD |
| COMERCIAL | CR | R | — | — |
| FINANCEIRO | R | RU | — | RU |
| GESTOR | R | R | — | — |
| AUDITOR | R | R | — | — |

**Legend:** C=Create, R=Read, U=Update, D=Delete

## Acceptance Criteria

- [ ] `permissions` table seeded with full matrix
- [ ] `@CheckPermission('sales', 'CREATE')` decorator works
- [ ] Middleware validates permission before controller executes
- [ ] Unauthorized access returns HTTP 403
- [ ] Dynamic permission grant/revoke (ADMIN only)
- [ ] Tests verify role boundaries (COMERCIAL can't POST /users)

## Implementation

1. **Database:** Seed permissions matrix via migration
2. **Guard:** `PermissionGuard` + `@CheckPermission()` decorator
3. **Service:** Permission checking logic
4. **Tests:** Unit tests for each role/resource combination
