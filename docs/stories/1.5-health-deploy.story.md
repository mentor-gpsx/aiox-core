# Story 1.5: Backend Health Check + Deployment Setup

## Story Statement

Polish backend with health checks, structured logging, error handling middleware, and deployment configuration for Vercel/Railway/Render.

## Acceptance Criteria

- [ ] `GET /health` returns 200 + `{ status: 'ok' }`
- [ ] `GET /api/status` includes DB connection status
- [ ] Environment variables validated at startup (error if missing)
- [ ] Graceful shutdown (10s timeout)
- [ ] Structured logging (Winston or Pino)
- [ ] Error middleware returns JSON 500s with requestId
- [ ] CORS configured for frontend URL
- [ ] Rate limiting optional but recommended
- [ ] Deployment script for Vercel/Railway/Render
- [ ] README documents env vars + deployment steps

## Implementation

1. **Health Service:** Already implemented in Story 1.4 ✅
2. **Error Filter:** Custom exception filter for JSON responses
3. **Logging Middleware:** Request/response logging
4. **Graceful Shutdown:** Handle SIGTERM for clean exits
5. **Deployment:** Config files for chosen platform
