# Story 1.1: Implement Supabase Auth with Sign Up / Login / Logout

## Story Statement

Implement complete authentication system using Supabase Auth + JWT tokens. Users must be able to register, login, logout, and maintain sessions. This story **unlocks Story 1.2+ (RLS depends on auth users)**.

## Acceptance Criteria

### Backend Auth
- [x] Supabase Auth enabled (Email/Password)
- [x] POST `/auth/signup` — register user (email + password)
- [x] POST `/auth/login` — authenticate (returns JWT + user data)
- [x] POST `/auth/logout` — invalidate session
- [x] JWT payload includes: user.id, user.role, exp
- [x] CORS configured for frontend origin

### Frontend Auth UI
- [ ] Login page (email + password form)
- [ ] Logout button in sidebar
- [ ] Protected routes redirect to login if unauthenticated
- [ ] Session persists in localStorage (JWT token)

### Tests
- [x] Unit tests: signup, login validation (edge cases)
- [x] E2E tests: login flow (happy path + error cases)
- [x] At least 80% coverage

## Implementation Strategy

1. **Database:** Create `users` table with email, password_hash, role, metadata
2. **Auth Service:** NestJS module with Supabase client for auth operations
3. **JWT Guard:** `@UseGuards(JwtAuthGuard)` decorator for protected routes
4. **Frontend:** React/Vue components for login form + session management
5. **Tests:** Jest unit + E2E tests covering auth flows
