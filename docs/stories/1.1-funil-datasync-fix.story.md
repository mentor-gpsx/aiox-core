# Story 1.1: Fix Funil de Vendas Data Sync: 98→2 leads loss

## Story Statement

The Funil de Vendas dashboard displays **only 2 leads** in Prospecção when **98+ should display**. Multi-agent investigation identified a **cascade of 15 compounding defects** across ClickUp fetch pipeline, API aggregation, and frontend rendering. Root cause: missing pagination, hardcoded frontend username filtering, and silent mock fallback masking failures.

**Impact:** Data appears corrupted in production. Users lose visibility of 96+ leads. Invalid business decisions due to incomplete pipeline view.

**Definition of Done:** `/api/funil-by-user` returns lead count **within ±5% of ClickUp UI** without silent failures or hardcoded filters.

---

## Context

### Problem Symptoms
- Dashboard shows **2 leads** when audit confirms **98+ exist** in ClickUp CRM OFICIAL
- Frontend hardcoded to only render 4 seller buckets (Maria, Nicolas, Kennyd, Gabriel)
- Any lead assigned to other users becomes **invisible**
- API errors silently return mock data (HTTP 200) instead of error status
- No `.env` loading — CLICKUP_API_KEY defaults to empty string

### Root Cause Analysis

**Cascade of defects (not single failure):**

1. **F1: Frontend Username Filter** — Only renders 4 hardcoded usernames via `.includes()` substring match. Any username not containing `maria|nicolas|kennyd|gabriel` → invisible accordion
2. **F2: No ClickUp Pagination** — API hard-caps at 100 tasks/page. No loop implemented. Tasks 101+ never fetched
3. **F3: Closed Tasks Excluded** — `include_closed=false` by default. "Pago"/completed deals missing
4. **F4: Silent Mock Fallback** — Any API error → returns mock data (HTTP 200) with exactly 2 users, masking failures
5. **F5: .env Not Loaded** — `require('dotenv')` missing. CLICKUP_API_KEY empty by default

**Additional defects (P1+):**
- F6: Custom field UUID lookup broken
- F7: Stage matching via task name substring (60-90% misclassification)
- F8: Pagination cap at 100 in loadFunilByUser
- F9: Multi-assignee duplication inflates count
- F10: API key hardcoded in public HTML (security)
- F11: Env var name mismatch
- F12: RLS N+1 queries
- F13: Schema missing 4 critical columns

   ```javascript

     allTasks.push(...response.tasks);
     if (response.tasks.length < 100) break;
   }
   ```
